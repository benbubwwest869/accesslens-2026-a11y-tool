## What AccessLens Does

AccessLens is built for teams that want to evaluate interface ideas through an accessibility lens before implementation begins. It is aimed at UI design review, giving product, design, and engineering groups a quicker way to identify issues that could influence usability and compliance.

Because it checks designs against WCAG 2.1 AA expectations, AccessLens helps teams make accessibility decisions earlier in the process. That makes it a practical fit for web-first interfaces and HTML-based design assets where feedback needs to happen before code is written.

## How to Use It

1. Open AccessLens in your browser.

3. Start an accessibility review on the design.
4. Review results tied to WCAG 2.1 AA concerns.
5. Make the needed updates before development moves ahead.

- Check for accessibility issues
- Adjust the design
- Re-run the review
- Share the results with your team
